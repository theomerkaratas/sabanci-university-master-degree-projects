Concurrency

Scale the application by adding more independent processes rather than making a single process larger.

  • The Process Model: Assign different tasks to different process types. For example, HTTP requests are handled by a web process, while long-running background tasks are handled by a worker process.
  • Horizontal Scaling: Because processes are stateless and share-nothing (Factor VI), scaling is as simple as spinning up more copies of a specific process type. This allows the app to handle increased load effortlessly.
  • Individual Management: Different process types can be scaled independently. If you have a backlog of background jobs, you can scale the worker processes without needing to add more web processes.
