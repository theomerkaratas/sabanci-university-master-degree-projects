Disposability

Maximize robustness with fast startup and graceful shutdown.

  • Fast Startup: Processes should strive to minimize startup time. This allows for more agile release cycles and moving processes across hardware quickly if a failure occurs.
  • Graceful Shutdown: Processes should shut down cleanly when they receive a SIGTERM signal. For a web process, this means refusing new requests but finishing the one it is currently processing before exiting.
  • Crash Resilience: The app should be designed to handle unexpected, non-graceful terminations. By being stateless and using robust queuing systems for workers, the app remains stable even if a process dies suddenly.
