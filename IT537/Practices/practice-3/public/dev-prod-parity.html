Dev/Prod Parity

Keep development, staging, and production as similar as possible to minimize friction and bugs.

  • The Three Gaps: Traditionally, there are gaps in time (weeks between deploys), personnel (devs write code, ops deploy it), and tools (using SQLite locally but Postgres in prod). A twelve-factor app closes these gaps.
  • Tooling Consistency: Developers should use the same backing services in their local environment as are used in production. Using "lightweight" versions of databases locally often leads to subtle, hard-to-track bugs.
  • Continuous Deployment: By keeping environments identical, developers can feel confident deploying code minutes after it is written, leading to a faster and more agile development cycle.
