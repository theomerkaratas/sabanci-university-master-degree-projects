Logs

Treat logs as event streams rather than files stored on a disk.

  • Standard Output: The application should not concern itself with routing or storing its own logs. It should simply write its event stream to stdout (standard output).
  • Environment Handling: In development, the dev sees the stream in their terminal. In production, the execution environment captures these streams and routes them to a final destination, such as a log-indexing system like ELK or Splunk.
  • Introspection: Treating logs as a continuous stream allows for real-time analysis, alerting, and long-term storage without the app needing to manage log rotation or file permissions.
