Build, Release, Run

The transformation of a codebase into a running deployment happens in three strictly separated stages.

  • The Build Stage: This stage transforms the code into an executable bundle known as a build. It fetches dependencies and compiles binaries/assets.
  • The Release Stage: This takes the build and combines it with the current deployment's configuration. The resulting release contains both the executable and the settings, ready for immediate execution.
  • The Run Stage: This is where the app actually runs in the execution environment. This stage should be kept as simple as possible, as errors in the run stage cannot be easily fixed without creating a new release.
