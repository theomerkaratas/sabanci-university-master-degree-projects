Config

An app's configuration is everything that is likely to vary between deployments (staging, production, developer environments).

  • The Environment Rule: Store configuration in environment variables. Unlike config files, environment variables are unlikely to be checked into code repos accidentally and are easy to change between different deploys without changing code.
  • The Open Source Test: A great litmus test for whether your config is handled correctly is asking: "Could the codebase be made open source at this very moment without revealing any credentials?" If not, your config is improperly hardcoded.
  • Strict Separation: Config must be strictly separated from code. Code does not change between deploys, but config does. Grouping config into "environments" (like "development" vs "production") does not scale as well as managing individual environment variables.
