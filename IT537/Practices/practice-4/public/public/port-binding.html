Port Binding

A twelve-factor app is completely self-contained and does not rely on the runtime injection of a webserver into the execution environment.

  • Self-Contained Services: The web app exports HTTP as a service by binding to a port and listening for incoming requests. It "becomes" the server itself.
  • Protocol Agnostic: While HTTP is the most common, any service (like Redis or an XMPP server) can be exported via port binding. In development, the app is accessed via a URL like http://localhost:5000/.
  • Composition: Port binding means that one app can become the backing service for another app by providing the URL to the consumer app in its configuration.
