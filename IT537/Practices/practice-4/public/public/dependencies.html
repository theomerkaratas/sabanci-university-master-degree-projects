Dependencies

A twelve-factor app never relies on the implicit existence of system-wide packages or tools.

  • Explicit Declaration: You must use a dependency declaration manifest (such as a package.json, requirements.txt, or Gemfile) to list every library the app needs to function.
  • Isolation: Use a dependency isolation tool to ensure that no implicit dependencies "leak" in from the surrounding system. This guarantees that the app behaves the same way regardless of the machine it is running on.
  • No System Tool Reliance: If the app needs to shell out to a system tool (like curl or imagemagick), that tool should be treated as a dependency and bundled into the app or installed via a controlled setup script.
